Основные правила по СSS, для проверки

  1. Не забудь сбросить дефолтные стили
  2. Не используй названия цветов, лучше использовать rgb (подходит для старых браузеров), rgba (не для старых браузеров). Как выход используй оба правила цвета.
  3. Используй безопасные цвета (хотя это в прошлом…)
  4. Background-image только для красоты, а не для структуры
  5. Сокращай записи для слойств (margin, pading, border, background, font)
  6. Порядок псевдоклассов важен
  7. Если браузер располагает блоки по вертикали, то их поля будут равны большему из этих двух элементов
  8. Изпользуй float для плавающих изображений

Типографика

  1. Загружай шрифт @font-face(если шрифт доступен на сервере, его проще контролировать используй это для важных проектов, так как можно полностью контролировать форматы и совместимость со старыми браузерами ). или google-fonts или Adobe Fonts (этот лучше использовать для своих личных проектов, а также для проектов портфолио, либо если подразумеваешь, что сайт будет читать формат WOFF2)
  2. Загружай каждое начертание шрифта отдельными правилами @font-face. Предустановленные шрифты автоматически подберут правильное начертаний, а вот для загружаемых начертаний нужно использовать отдельные @font-face
  3. Если мы думаем про IE8 и более ранние версии, то создаем те же 4 правила, вот только названия меняем...А в свойствах к нужному селектору прописываем отдельный font-family, font-style, font-weight (последние 2 указывай обязательно, так как некоторые браузеры будут стараться уплотнять либо наклонять начертание).
  4. Указывай несколько видов шрифтов для семейства и заканчивай шрифтом по умолчанию (serif, sant-serif, monospase и тп)
  5. Для более быстрой работы лучше использовать шрифты не требующих загрузки, то есть те что предустановлены. Какие шрифты установлены на ОС можно посмотреть на данном сайте
  6. Шрифты можно подправить и выгрузить используя Webfont Generator.
  7. Для основного текста испотзуй шрифты с разными начертаниями, а для заголовков можешь брать только с одним начертанием
  8. Лучше всего для body указать размер шрифта ключевым словом (если только не пользуешься google fonts, там лучше указывать цифрами 400 - normal, bon), а для остальных селекторов лучше использовать em или %.(проверь на глубокой вложености) Старые версии эксплоера не поддерживают масштабирование, если размер шрифта задается в пикселях.
  9. Насыщенность шрифта указывай ключевыми словами
  10. Если мы добавили новый шрифт и хотим, чтобы теги типа em и strong, также изменять текст на курсив и жирный, то нужно добавить еще несколько правил @font-face с таким же именем семейства но разными свойствами font-weight и font-style, а также конкретным шрифтом который мы для этого дела загружаем. При таком подходе пропадает необходимость дописывать новые свойства отдельным селекторам.
  11. Для указания размера шрифта лучше использовать em и rem (не для IE8 и младше).
  12. Лучше не подчеркивать текст не являющийся ссылками
  13. Лучше не использовать белый текст на белом фоне обозначенный лишь текстом. Так как на старом браузере такой текст не отобразиться
  14. line-height лучше прописывать числовое значение без единиц измерения
